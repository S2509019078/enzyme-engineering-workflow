# 酶改造可复用工作流设计

## 目标

建立一个面向 Windows PowerShell 与 Python 的酶改造候选筛选仓库。用户在新项目中只需替换 FASTA、结构文件、EVcouplings/ESM 类结果和配置，即可完成候选突变标准化、序列—结构映射、FoldX 批量扫描、断点续跑、汇总和作图。

## 支持范围

主流程包括：输入检查、在线工具结果导入与标准化、候选位点筛选、每个位点 19 种替换生成、FASTA 与 PDB 残基映射、FoldX RepairPDB/BuildModel 调度、长任务断点续跑、结果解析、突变名称还原、汇总表和网页风格热图。

ESM-2 与 EVcouplings 的在线分析本身用可复现操作指南记录，因为其服务端实现和输出可能变化；仓库不伪装成完全本地重现这些网站。FoldX 程序和许可证不随仓库分发。

## 用户接口

用户复制示例项目后维护：

- `config/config.yaml`：FoldX 路径、运行次数、批量策略、阈值、输出位置和绘图设置。
- `config/enzymes.tsv`：酶名、FASTA 路径、PDB 路径、结构链和可选在线结果路径。
- `inputs/`：序列、结构和在线工具导出的 CSV。

主入口支持环境检查、标准化在线结果、验证位点映射、准备 FoldX、运行批量扫描、仅重建汇总/热图和查看状态。软件路径不写死在 Python 文件中。

## 组件边界

1. 输入审计器检查 FASTA、PDB、链、残基编号、野生型氨基酸和 CSV 列结构。
2. 在线结果适配器把不同列名与记分方向转成统一表，并保留原始分数和来源，避免混淆 EV 分数与 FoldX ΔΔG。
3. 位点映射器通过序列比对建立 FASTA 位点到 PDB 链/残基号的显式映射；无法唯一映射的位点不进入 FoldX。
4. 突变生成器按配置产生候选列表，排除野生型自身、非标准残基和明确禁止位点。
5. FoldX 调度器负责 RepairPDB 与 BuildModel，按酶分目录保存状态，支持中断后继续和已有结果完整性判断。
6. 解析器读取 FoldX 输出，将结构编号还原为用户序列编号，并生成统一的 `mutation`、`ddg_kcal_mol`、`status`、`source` 字段。
7. 报告器生成全量表、稳定/中性/不稳定分类、每酶摘要、热图和运行状态报告。

## 数据流与目录

原始输入进入 `inputs/` 且保持不变；标准化表与映射表进入 `work/normalized/` 和 `work/mappings/`；FoldX 原始输出进入 `work/foldx/`；最终 CSV/XLSX、图和摘要进入 `results/`。报告可从已有 FoldX 原始结果独立重建，不要求重新计算。

## 长任务与错误处理

每个酶和每批突变都有状态记录。状态日志写入失败不得终止 FoldX 主计算：程序将重试并写入备用日志。脚本启动时识别完整结果、部分结果和损坏结果，只跳过完整任务。单个突变失败会被记录，其他任务继续。预计任务量和粗略耗时在执行前显示。

## 科学解释边界

EVcouplings/ESM 类分数用于序列层候选优先级，FoldX ΔΔG 用于结构稳定性估计，两者不能互相替代。仓库输出的是计算候选而非实验验证结论；README 说明催化位点、保守位点和稳定性筛选的联合解释方式，并要求最终进行实验验证。

## 验证

使用小型模拟 FASTA、PDB、在线结果和 FoldX 输出测试列名适配、序列映射、19 种替换生成、命名还原、断点识别、结果解析和热图矩阵。Python 代码进行语法检查和单元测试；真实 FoldX 执行需由用户在已授权环境中完成。

## 发布边界

仓库不包含原始 JSON、个人目录、在线账户信息、FoldX 二进制/许可证、大型结构或真实批量结果。`.gitignore` 排除本机配置、日志、缓存和计算输出，只提交示例与小型测试数据。
